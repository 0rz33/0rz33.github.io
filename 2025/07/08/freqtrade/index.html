技术指标 币种特性

核心区指标不变

  1. 核心趋势指标
    • EMA组合(10/50/100周期)
    • MACD(12/26/9参数) 金叉死叉
    • ADX趋势强度(14周期)
    • 多均线交叉信号(ema_cross)
  2. 动量震荡指标
    • RSI(14周期)
    • 随机指标(14,3,3)
    • CCI(14周期)
    • 价格变化率(return)
  3. 波动率管理指标
    • 布林带(20周期，2σ)
    • ATR真实波幅(14周期)
    • 布林带宽度特征(bb_width)
    • 价格波动率(14周期标准差)
  4. 成交量特征
    • 成交量Z分数(volume_zscore)
    • 成交量均值(14周期×1.5乘数)
    • VWAP(20周期)
  5. 特殊优化指标

持仓管理

flowchart TD
    A[获取当前账户状态\n（余额、持仓、价格、手续费）] --> B[判断市场信号\n（是否开仓、加仓、平仓）]

    B -->|开仓信号| C[是否已有持仓？]
    B -->|无信号或持仓中| Z[等待下一信号]

    C -->|已有持仓| D[判断是否满足加仓条件]
    C -->|无持仓| E[计算开仓资金与风险\n（资金占比、最大亏损等）]

    D --> F[计算加仓资金与风险\n（资金投入、风险敞口）]
    E --> G[执行开仓下单]

    F --> H[执行加仓下单]
    G --> I[更新持仓记录]
    H --> I

    I --> J[止盈 / 止损 / 换仓判断]
    J --> K[是否满足平仓条件？]
    K -->|是| L[计算当前盈亏与风险比\n（盈亏率、RR比、最大回撤）]
    K -->|否| Z

    L --> M[执行平仓 / 止损 / 减仓下单]
    M --> N[更新资金与交易日志]

xgboost心得

  1. 注意币种特性
  2. 用xgboots分析市场入场信号

    • 权重必不可少 因为交易都有冲突
  3. 入场后，交易中，离场
    • 动态止顺止盈，持仓情况
    • 风控要做好
    • 权重必不可少 因为交易都有冲突
    • 模型可以随时调整
    • 多指标融合 不同策略
  4. xgboost训练数据
    • 训练加入监控
    • 调整相应参数

  1. 入场流程
flowchart TD
    %% === 获取账户与环境状态 ===
    A[📊 获取账户状态
（余额、持仓、手续费、浮盈/浮亏）] --> A0[🧭 判断当前市场行情类型 → 设置 market_state]
    A0 --> S1[📌 策略风格选择
趋势 / 震荡 / 极端]

    %% === 判断是否持仓 ===
    S1 --> B{📦 是否已有持仓？}

    %% === 空仓逻辑：构建信号 & 开仓 ===
    B -- 否 --> C1[📈 扫描币种信号]
    C1 --> C2[🔌 加载信号强度插件
（按策略调用指标/ML模型）]
    C2 --> C3[✅ 筛选符合入场标准的币种]
    C3 --> C4[⚙️ 获取基础权重配置
（预设 + 用户偏好）]
    C4 --> C5[🧮 动态权重 = 基础权重 × 信号强度 × 波动率因子 × 资金流因子]
    C5 --> C6[📏 权重归一化（∑=1）]
    C6 --> C7[💰 资金分配（按权重）]

    C7 --> C8[🔍 检查是否 ≥ 最小下单金额]
    C8 -- 否 --> X1[🚫 跳过币种]
    C8 -- 是 --> C9[🔐 风控检查（最大仓位 / 动态限额）]

    C9 -- 否 --> X2[🚫 跳过币种]
    C9 -- 是 --> C10[🛒 执行建仓下单]
    C10 --> C11[📥 更新持仓记录]
    C11 --> D1[📡 进入持仓监控模块]

    X1 --> C3
    X2 --> C3

    %% === 持仓逻辑：监控与调整 ===
    B -- 是 --> D1

    D1 --> D1a[🛡️ 风控检测
（仓位超标 / 市场剧变 / 单币风险）]
    D1a -- 触发 --> E4[⚠️ 风险事件处理
（强制减仓 / 平仓）]
    D1a -- 正常 --> D2{是否触发止盈？}

    D2 -- 是 --> E1[✅ 平仓止盈 → 更新持仓记录]
    D2 -- 否 --> D3{是否触发止损？}
    D3 -- 是 --> E2[✅ 平仓止损 → 更新持仓记录]

    D3 -- 否 --> D4{是否满足加仓 / 调仓？}
    D4 -- 是 --> E3[🔁 调整持仓 → 更新记录]
    D4 -- 否 --> D5[🧭 使用 market_state 适配策略风格]

    D5 --> F1[📌 当前为趋势行情 → 启用趋势管理策略]
    D5 --> F2[📌 当前为震荡行情 → 启用区间持仓策略]
    D5 --> F3[📌 当前为极端行情 → 启用反弹 / 避险策略]

    F1 --> D1
    F2 --> D1
    F3 --> D1
模块 功能
🧭 市场判断 判断当前行情类型，指导信号模块及策略选择
🔌 信号模块 插件式结构，支持按策略动态切换：指标、模型、复合打分
🧮 动态权重计算 多因子融合（信号 × 波动率 × 资金流 × 用户偏好）
🛒 下单建仓 资金分配 + 最小单限制 + 风控检查
📡 持仓监控 止盈止损、加仓调仓，复用行情判断逻辑
📌 策略模块 根据趋势 / 震荡 / 极端，执行不同策略逻辑（调仓 / 继续持有 / 反手）

市场状态枚举

根据市场状态 ✅ 分类逻辑总览

状态 优化后的判断逻辑（综合指标特征）
STRONG_TREND_UP 多头排列 (ema_10 > ema_50 > ema_100) 且 ema_cross == 1 且 macd > 0 且 adx > 25 且 trend_strength > 80 且 rsi > 60 且 volume_score > 50
TREND_UP ema_10 > ema_50 且 macd > 0 且 adx > 20 且 trend_strength > 60 且 rsi > 50，不满足强趋势条件时适用
WEAK_TREND_UP ema_10 ≥ ema_50 且 macd_hist > 0 且 trend_strength > 40 且 50 ≤ rsi ≤ 60 且 EMA排列非多头
RANGING EMA排列混乱 (chaotic)，且 bb_width < 0.02 且 adx < 15 且 trend_strength < 30 且 consolidation_ratio > 70
WEAK_TREND_DOWN ema_10 ≤ ema_50 且 macd_hist < 0 且 trend_strength > 40 且 40 ≤ rsi ≤ 50 且 EMA排列非空头
TREND_DOWN ema_10 < ema_50 且 macd < 0 且 adx > 20 且 trend_strength > 60 且 rsi < 50
STRONG_TREND_DOWN 空头排列 (ema_10 < ema_50 < ema_100) 且 ema_cross == -1 且 macd < 0 且 adx > 25 且 trend_strength > 80 且 rsi < 40 且 volume_score > 50
BREAKOUT breakout_strength > 70 且 bb_width 短时内快速放大（可用 bb_width.diff() > threshold 判断） 且 atr_percent > 3 且 volume_breakout > 60
REVERSAL rsi > 70 or rsi < 30 极端值，且 momentum_score < 20 且 macd_hist 由正变负 / 负变正（方向反转）
UNKNOWN 不满足以上任一状态逻辑
