MCP

1️⃣ 核心思路

  1. 选择开源模型
  • 小模型（CPU/单卡可跑）：GPT4All, LLaMA-7B, MPT-7B
  • 大模型（多卡 / 大显存）：LLaMA-13B, Falcon-40B, MPT-30B
  1. 选择推理框架
  • Ollama → REST / CLI
  • Transformers + PyTorch / Accelerate → 官方框架，灵活
  • vLLM → 高效推理，大批量请求性能好
  • Text-generation-webui → 快速部署 REST / WebSocket API
  • FastChat → 聊天模型专用，支持多轮对话
  1. 提供 REST / SSE 接口
  • 使用 FastAPI 封装 MCP 接口
  • /v1/chat/completions 普通返回
  • /v1/chat/stream 流式输出

与 MCP 框架的对比

特性 Ollama vLLM / WebUI / Transformers
本地部署 ✅ ✅
云端调用 ✅ ❌（需自己部署）
API 支持 REST / CLI REST / SSE（自定义）
流式输出 ✅ ✅
模型自由度 受限 完全自由，开源模型都可用
免费 / 开源 部分免费 完全开源

2️⃣ 部署示例（FastAPI + Transformers）

3️⃣ 部署建议

  1. 硬件要求
    模型 < 7B → 16GB 显存或 32GB 内存可跑
    模型 13B → 40GB 显存或多卡
    模型 30B+ → 80GB+ 显存或分布式 GPU

  2. 优化推理
    使用 torch_dtype=torch.float16 或 load_in_8bit=True 降低显存
    用 vLLM 或 Text-generation-webui 可提高并发性能

  3. 多模型支持
    MCP 可扩展多个模型
    请求时通过 model 字段选择模型

  4. 安全性
    对外提供 API 时，可加 Token 校验
    限制单次请求 max_tokens 或并发量
