Sealos

Sealos -> CasaOS -> Home Assistant

SeaOS 是一个专为家庭设计的中控系统（Home Control System / Smart Home Hub），核心目标是把家里的各种智能设备、存储、网络服务整合在一个平台上，实现统一管理和自动化

  • 跨设备支持：可以控制智能灯、智能门锁、摄像头、传感器等。
  • 家庭自动化：支持条件触发、场景设置、定时任务。
  • 本地化存储：可与 NAS 或家庭服务器结合，避免云端依赖。
  • 开放接口：可以与第三方系统或自建程序对接。

初级AI管家

功能要求

外出/在家场景

  1. HomeAssistant 控制室内智能家电
  2. 手机容量不足 可以存储图片在家里
  3. 可云办公 上传下载
  4. 监控植物系统
  5. (在家场景)可呼叫语音助手

硬件需求

树莓派 4B

  • 系统： RPi OS

  • 核心功能：

    • HomeAssistant（家庭自动化核心中枢）
    • MQTT / Zigbee2MQTT 网关
    • 本地语音助手（控制家居 + 调用 AI）
    • Nextcloud（轻量云盘 → 云办公 / 文件同步）
    • AI 前端代理：将语音/文本请求转发到 PC 工作站 AI 模型
    • 控制全家 Zigbee / WiFi 智能家电
    • 外出时通过 VPN 访问树莓派，统一管理家庭设备

  • 需要外壳（配件）：

    • 3D 打印外壳：JVC3100r
    • 显示触摸屏 + 键盘｜（AI 管家界面）
    • pi4 respeaker（声控） + USB 音频（声控 喇叭 ）｜ 已具备：喇叭 + 麦克风（用于语音助手）

PC AI 工作站

  • 系统： Linux / Windows（推荐 Linux）
  • 核心功能：
    • GPU 推理（大模型计算核心）
    • 托管私有大模型（LLM / Vision / TTS / ASR / RVC）
    • 接收树莓派发来的 AI API 请求
    • 为 HomeAssistant 提供视觉识别、自动化推理
    • 家庭主要大文件存储区：
      • 素材
      • 数据
      • AI 模型
      • 多媒体文件
    • 高性能 Worker：处理 AI 任务
    • 可被 HomeAssistant 调用执行识图、识物、自动化联动
  • 需要外壳（配件）：
    • GP02A半身像1
    • GP02A半身像2
    • 最好能抽拉出主机
    • ITX主板
  • 硬件需求：
    • 强力 GPU（RTX 4070 / 4080 / 4090 或更高）
    • 至少 32GB RAM
    • NVMe SSD（1TB–2TB，模型加载更快）
    • 良好散热系统
    • 稳定供电

架构

步骤指南

1️⃣ 树莓派 4B —— 家庭中控

  1. 硬件组装

    • 使用 JVC3100r 3D 打印外壳
    • 安装小型显示屏（作为 AI 管家界面）
    • 接入喇叭 + 麦克风（语音助手）
    • 可选：风扇 / 铜片散热

  2. 系统安装

    • 安装 RPi OS（64 位版本，推荐 Raspberry Pi OS Lite + GUI 可选）
    • 配置网络：静态 IP / DHCP 保留，确保在家庭内网可访问
    • 有键盘和显示屏

  3. 软件安装

    • Docker & Docker Compose
    • HomeAssistant 容器：
      • 核心自动化中枢
      • 配置 Zigbee / MQTT 网关
    • Nextcloud 容器：
      • 轻量云盘，实现文件同步 / 云办公
    • Rhasspy / OpenVoiceOS：
      • 本地语音助手 = Home Assistant + ReSpeaker + Rhasspy + Vosk
      • 可调用 AI 服务
    • Mosquitto (MQTT broker)
    • Zigbee2MQTT（控制 Zigbee 设备）
    • VPN 客户端：
      • Tailscale / WireGuard / Zerotier，用于外网访问

  4. AI 前端代理

    • 配置树莓派将语音/文本请求通过 API 转发给 PC AI 工作站
    • 确保返回结果能被 HomeAssistant 或语音助手处理

  5. 智能家居集成

    • 配置所有 Zigbee / WiFi 设备
    • 创建基础自动化规则
    • 外出时可通过手机 / 平板 / 笔电控制

2️⃣ PC AI 工作站 —— AI / 计算中心

  1. 硬件组装

    • 使用半身像机箱（GP02A 或可抽拉式 ITX 机箱）
      • GP02A半身像1
      • GP02A半身像2
    • 安装强力 GPU（如 RTX 4070/4080/4090）
    • 配置 NVMe SSD（1–2TB）
    • 至少 32GB RAM
    • 良好散热 + 稳定电源

  2. 系统安装

    • Linux（Ubuntu / Pop!_OS / Debian 推荐）
    • 安装 GPU 驱动 + CUDA / cuDNN（支持 AI 推理）
    • 配置静态 IP（家庭内网访问）

  3. 软件安装

    • Docker & Docker Compose
    • AI 模型部署：
      • LLM（如 LLaMA、MPT、本地 Whisper）
      • Vision 模型（图像识别 / 视频分析）
      • TTS / RVC / ASR 模型
    • REST / WebSocket API 服务：
      • 接收树莓派发来的请求
      • 返回 AI 推理结果
    • 可选：VNC / RDP，用于远程桌面管理

  4. 与树莓派集成

    • 配置树莓派作为前端：
      • API 请求通过家庭内网发送
      • 结果回传到树莓派供 HomeAssistant 或语音助手使用
    • 监控 AI 任务负载
    • 存储大文件、素材、模型权重

3️⃣ 外网访问与安全

  1. VPN 设置

    • 在树莓派或路由器端配置 Tailscale / WireGuard
    • 手机 / 笔电通过 VPN 进入家庭内网
    • 访问 HomeAssistant、Nextcloud、AI 服务

  2. 文件访问

    • Nextcloud 提供 SFTP / WebDAV 接口
    • 上传、下载、备份工作文件
    • 支持外出办公

  3. 智能家居控制

    • HomeAssistant WebUI 或手机 App
    • 可控制灯光、空调、门锁、植物监控系统
    • 触发自动化场景

4️⃣ 流程概览

  1. 手机 / 笔电 → VPN → 树莓派
  2. 树莓派：
    • 处理 HomeAssistant 控制
    • Nextcloud 文件同步
    • AI 请求转发 → PC 工作站
  3. PC 工作站：
    • GPU 进行 AI 推理
    • 返回结果给树莓派
  4. 树莓派执行智能家居或语音助手动作
  5. 外出可上传 / 下载重要文件，远程管理家居

✅ 这样就形成了一个完整的家庭 AI 管家系统：

  • 树莓派 = 前端中控 + 轻量云盘 + HomeAssistant
  • PC 工作站 = 后端 AI 核心 + 高性能存储
  • VPN = 安全远程访问
  • Nextcloud = 云办公 / 文件同步
  • HomeAssistant + Zigbee / MQTT = 智能家居控制

一、语音验证（说话人识别）

方案 方法 优点 适用场景
Vosk + X-vector 离线语音验证 轻量级，树莓派可运行 本地语音助手
DeepSpeaker (D-vector) 深度学习模型 高准确率 复杂应用，如智能门禁
ECAPA-TDNN SOTA 声纹识别 最新算法，高精度 多用户识别

二、语音唤醒（Wake Word Detection）

  • 需要一个始终在监听的“唤醒词”（如 Siri 监听“Hey Siri”）。
  • 解决方案：
    • Porcupine（轻量级，支持离线）
    • Snowboy（不再维护，但仍可用）
    • Mycroft Precise（开源，可训练自定义唤醒词）
    • Vosk + KeywordSpotter → 自定义关键词检测

三、自然语言理解（Natural Language Understanding, NLU）

  • 解析用户的意图，比如“今天天气怎么样？” → 解析出“查询天气”。
  • 方案：
    • Rasa NLU（开源，可离线运行）
    • Snips NLU（轻量级，但 Snips 已被 Sonos 收购）
    • OpenAI GPT-4（需要联网，但效果强大）

四、工作流程

硬件需求

  • 树莓派 4B / 5
    • 推荐至少 4GB RAM，运行 ASR + 说话人识别时比较流畅。
  • 麦克风
    • USB 麦克风（简单易用，例如 TONOR TC-777）
    • 或 麦克风阵列（推荐 ReSpeaker 2/4/6-Mic Array for Raspberry Pi，带有降噪/回声消除）
    • 如果对说话人识别准确率要求高，阵列效果更好。
  • 存储
    • 至少 32GB microSD
    • 或者外接 SSD，ASR 模型会比较大。
  • （可选）扬声器/耳机：做语音交互时需要输出。

软件与模块对应

对应你流程里的各个步骤：

  1. 麦克风输入
    • 用 pyaudio / sounddevice 采集音频流。
    • 设置为 16kHz 单声道 PCM。
  2. Ring Buffer 缓存
    • 可以用 collections.deque 或者 sounddevice.RawInputStream 自带的回调缓存。
    • 缓存 1~3 秒的音频，保证唤醒词检测和语音片段提取。
  3. 唤醒词检测
    • 轻量：Porcupine（低资源占用，树莓派友好）
    • 可训练：Mycroft Precise
    • 自定义词：Vosk + KeywordSpotter
  4. 提取音频片段（1~2秒）
    • 当检测到唤醒词，立刻把 Ring Buffer 里的最后 1~2 秒 dump 出来保存为 .wav。
  5. 说话人识别（X-vector）
    • 依赖 pyannote.audio 或 SpeechBrain，输入音频片段，输出 speaker embedding → 与本地数据库比对。
    • 树莓派 4/5 能跑，但建议用轻量模型或外接 USB TPU（Coral USB Accelerator）。
  6. 身份确认
    • 如果 embedding 相似度 > 阈值（如 0.75），则确认说话人。
    • 否则忽略唤醒，避免被陌生人触发。
  7. 完整语音识别（ASR）
    • 用 Vosk（支持离线运行，中文/英文等都有预训练模型）。
    • Vosk-pi 版本可在树莓派直接跑，速度较慢，但可以用小模型（如 vosk-model-small-cn-0.3）。
