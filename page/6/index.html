导光器制作

反光杯 + 聚光透镜 + 光纤 = 高效导光器

反光杯：手电筒的灯胆
聚光透镜就是：一半的凸透镜
光纤：导光的

优势：光强高、光斑小、光纤耦合效率高
限制：光源功率、焦点热量、光纤耐热、精密对准
可以说这是 DIY 高效率光纤引光器的最佳组合之一，比单透镜或单反光杯效率高很多。

光伏储能混合系统（Hybrid PV System）

其实就是 “太阳能板 + 电池 + 混合逆变器 + 家用电网” 的综合解决方案。

所需硬件

核心部件

  • 光伏组件（Solar PV Modules / Solar Panels）
    作用：把太阳光转为直流电（DC）。
    要点：选单晶硅、高转换效率（≥19% 更优）、带 PID 抗衰能力。组件功率按屋顶面积/需求选择（如 300–540 W/panel）。
    数量 = 目标系统容量（kW） ÷ 单板功率（kW）。

  • 混合逆变器（Hybrid Inverter / PV Inverter with ESS）
    作用：DC→AC、MPPT、管理电池充放电、并网与离网切换。
    要点：支持双向充放电、并网认证、离网/备电输出、通信（RS485 / CAN / Wi-Fi）、峰值输出能力要匹配家中最大同时功率。
    常见规格：3kW / 5kW / 10kW 等；直流输入电压/最大 MPPT 电流要能接入你的 PV 阵列。

  • 储能电池（Battery Bank） — 推荐 LiFePO4（磷酸铁锂）
    作用：存储白天多余电能，用于夜间或停电时供电。
    要点：选择带 BMS 的电池包；标注循环寿命（≥3000 次@80% DOD 更好）；额定电压（48V 系统常见）或模块化 400V 系统（取决逆变器）。
    容量建议（家庭参考）：5 kWh / 10 kWh / 20 kWh。

  • 电池管理系统（BMS）
    作用：电池保护（过充、过放、均衡、温度保护）和与逆变器通讯。
    要点：必须与电池和逆变器兼容，支持报警与远程通信。

东南亚别墅建设与节能设计全攻略

地基阶段注意事项

防水处理

  • 关键节点加强防水：厨房、卫生间的地面与墙面交界处，务必采用聚氨酯或水泥基防水涂料，并向上返高≥30cm。
  • 设置防潮层：一楼地面下应预铺防潮膜（如PVC或自粘式沥青卷材），有效阻隔地气上升。
  • 户外地面坡度设计：屋外地坪应高于周边排水沟至少10cm，地面向外倾斜2~3%，防止雨水回流。

热水/供水布局

  • 热水管道预埋：采用PPR或PE-RT热水专用管，外包保温棉，防止热量流失。
  • 太阳能热水系统预留：
    • 上水、回水、溢水管道预埋；
    • 储水箱位置尽量设于高处以确保自然水压；
  • 泵系统智能化：
    • 安装断电自停功能，避免干烧；
    • 可选变频恒压泵以提升用水舒适度与节能效果。

防蚊虫叮咬

  • 高离地设计：将地面层抬高 40–80 cm，有效减少地面湿气与虫害滋生源。
  • 水沟与排水坡度设计：避免积水死角，是预防蚊虫滋生的核心设计。
  • 屋顶排水顺畅设计：设置排水沟槽、防堵落叶网，防止雨后积水孳生蚊子。
  • 无死角阴暗空间：减少蚊虫栖息区域，尤其在厨房、厕所、天花板边缘。

节能导向的地基设计

  • 保温层预留：在地坪施工前铺设XPS 挤塑板或泡沫混凝土，减少地面热传导；
  • 太阳能系统预布线：地基预留光伏电缆通道、电池房管线槽；
  • 雨水收集系统接口：预埋雨水排入储水池的管道，用于冲厕、浇花等二次利用。

水资源节能设计方案

级别 水质要求 用水场景 推荐水源
✅ 一级水 可直接饮用 饮水、做饭 自来水 / 反渗透（RO）处理后的雨水或地下水
✅ 二级水 可接触皮肤 洗澡、泳池、洗手 雨水+三级过滤杀菌 / 污水深度处理水
✅ 三级水 无皮肤接触 冲厕、洗衣机、室内清洁 雨水+普通过滤 / 灰水处理出水
✅ 四级水 非接触景观水 浇花、冲地、喷泉、景观池补水 荷花池雨水原水 / 简单过滤的灰水

🔧 每级水处理建议方案

  • 🥇 一级水（饮用）
    • 推荐：自来水 或
      • RO膜+UV消毒雨水系统
      • 末端设置净水龙头（饮水分开）
    • 用于：厨房饮用水、泡茶、做饭
  • 🥈 二级水（洗澡 / 泳池）
    • 雨水处理需：
      • 多级砂滤 + 活性炭 + 紫外线（或臭氧）
    • 池水需循环过滤系统 + 定期杀菌
  • 🥉 三级水（冲厕 / 洗衣）
    • 雨水+简单过滤即可；
    • 或使用处理后的灰水（洗澡水）
  • 🧯 四级水（浇花 / 景观）
    • 不需杀菌；
    • 可直接从荷花池抽取或雨水初滤水

🌧️ 雨水管理与回用

  • 屋顶雨水收集系统（落水管 + 滤网 + 蓄水桶）。
  • 灰水（洗澡/洗衣水）过滤后回用：冲厕、浇花。
  • 智能水表 + 水流实时监控，漏水报警。

🚿 节水设备与生活用水优化

  • 节水型洁具（双档马桶、节水花洒、起泡龙头）。
  • 可降解洗涤剂（适配灰水系统，避免堵塞/污染）。
  • 花园微滴灌系统，减少水耗、提高效率。

♻️ 污水处理与再利用

  • 处理方式：

    • 三格式化粪池：基础版，适用于1~3户。
    • 一体化小型污水处理设备：出水可灌溉/冲厕。
    • MBR 膜处理系统：可达中水标准，回用广泛。

  • 运行优化：

    • 使用电曝气或风曝气系统提升处理效率。
    • 搭配太阳能曝气机节能运行。

  • 回用建议：

    • 出水仅用于非饮用途，如景观、花园、厕所冲水。
    • 设置绿植过滤带/湿地吸收，提升生态安全性。

室内装修阶段注意事项

🚽 卫生间设计

  • 洗手台面应有微斜排水设计，避免水滞留。
  • 地面坡度 ≥1.5%，设置地漏 ≥2个，防止积水。
  • 建议做干湿分离（玻璃隔断/推拉门）。

🪟 窗户设计

  • 落地窗滑轮推荐安装于上方，更防尘易维护。
  • 建议使用中空双层玻璃 + 隔热断桥铝材料。
  • 窗户加装密封条，提升隔音与保温性能。

🍳 厨房设计

  • 选 ≥17m³/min 吸力油烟机，配备止回阀。
  • 抽油烟口向外，带防虫防雨罩。
  • 橱柜下方设计通风口或百叶门，防潮防霉。

照明与配电系统设计

配电安全

  • 强电/弱电分离布线，避免干扰，布线使用阻燃线管。
  • 所有空调、热水器、厨房电器应设置独立回路 + 空气开关；
  • 建议安装漏电保护器（ELCB）+ 电涌保护器（SPD），应对雷击等风险。

照明布局

  • 主照明建议使用LED 吸顶灯或筒灯，高效节能；
  • 卧室、走廊、卫生间建议搭配感应灯或夜灯，便捷又节能；
  • 考虑预埋智能照明控制系统（如 Zigbee、Wi-Fi、KNX），后期可接入 Home Assistant 等智能中枢。

🌿 节能建筑设计辅助方案

🍃 通风与遮阳设计

  • 中庭通风井、开窗对流、屋檐挑出 ≥0.8m。
  • 百叶窗与高位排气窗配合，增强排热与遮阳效果。

🧱 材料与结构选择

  • 使用本地天然建材（竹、柚木、红土砖），低碳环保。
  • 坡屋顶利于排水与热对流。
  • 设置独立“设备房”：集中布置电池、泵、过滤设备等，方便维护与检修。

✅ 附加建议（施工实用）

  • 插座预留合理，避免后期拖线。
  • 弱电布线提前规划：摄像头、Wi-Fi、门禁等。
  • 屋顶可加绿植或设凉亭，降温又美观。
  • 建议双管排水（黑水/灰水分离），提升回用系统效率。

生态鱼缸系统总结与维护要点

这个系统之所以能接近“不用护理”的状态，核心在于构建了一个功能完备的食物链和分解链，实现了养分在系统内部的良性循环：

核心优势：完整的生态循环链

  • 生产者：

    • 水草和睡莲利用阳光和鱼的粪便生长。

  • 初级消费者：

    • 鱼和虾吃藻类和水草（少量）。

  • 分解者（关键）：

    • 黑壳虾是吃死鱼、死虾和死蚬的“主力军”，防止尸体腐烂污染水质。
    • 蚬滤食水中的微生物和碎屑。
    • 细菌最终分解所有有机物。

  • 养分循环：

    • 植物吸收分解后的废物，水质保持清澈，形成良性循环。

还需要注意什么？

虽然系统设计得很完善，但要实现完全“不用护理”是不可能的。您依然需要进行最基础的维护：

  • 光照管理：

    • 阳光充足的南向房是优势，但需要注意遮阳，避免阳光直射时间过长导致夏天水温过高或藻类大爆发。您可能需要拉一下窗帘。

  • 水量补充：

    • 瀑布蒸发会带走水分，需要定期（比如每周）补充蒸发掉的水分。

  • 生物密度控制：

    • 如果鱼和虾繁殖过多，系统可能会不堪重负，需要适度捕捞或移出部分生物。

  • 偶尔的清理：

    • 睡莲枯叶、堆积过多的螺壳（如前所述）等，偶尔需要手动清除一下。

结论：

这个配置方案非常棒，只要您控制好生物数量和光照，它确实能实现极低维护的状态，是您想要的“生态鱼缸”的理想模型。

整体生态结构

阳台分区规划

🅐 厨余处理区（源头）

  • 密封式堆肥桶（防虫防臭）
  • 发酵液肥瓶（带放气）

🅑 土培区（主产出）

  • 花盆 / 种植箱
  • 使用自制堆肥 + 蛋壳粉

🅒 水培区（吸收营养水）

  • 静态水培瓶 / PVC 水培槽
  • 直接使用鱼缸换水

🅓 鱼缸区（生态核心）

  • 鱼缸 + 过滤 + 水草
  • 稳定供给氮源

基础步骤

  • （1）水培槽准备

  • （4）光照系统
    • LED生长灯（红蓝光）
    • 光照周期：
      • 叶菜类：12–16小时/天
      • 可用智能插座定时控制

SeaOS 是一个专为家庭设计的中控系统（Home Control System / Smart Home Hub），核心目标是把家里的各种智能设备、存储、网络服务整合在一个平台上，实现统一管理和自动化

  • 跨设备支持：可以控制智能灯、智能门锁、摄像头、传感器等。
  • 家庭自动化：支持条件触发、场景设置、定时任务。
  • 本地化存储：可与 NAS 或家庭服务器结合，避免云端依赖。
  • 开放接口：可以与第三方系统或自建程序对接。

一、语音验证（说话人识别）

方案 方法 优点 适用场景

二、语音唤醒（Wake Word Detection）

  • 需要一个始终在监听的“唤醒词”（如 Siri 监听“Hey Siri”）。
  • 解决方案：
    • Porcupine（轻量级，支持离线）
    • Snowboy（不再维护，但仍可用）
    • Mycroft Precise（开源，可训练自定义唤醒词）
    • Vosk + KeywordSpotter → 自定义关键词检测

三、自然语言理解（Natural Language Understanding, NLU）

  • 解析用户的意图，比如“今天天气怎么样？” → 解析出“查询天气”。
  • 方案：
    • Rasa NLU（开源，可离线运行）
    • Snips NLU（轻量级，但 Snips 已被 Sonos 收购）
    • OpenAI GPT-4（需要联网，但效果强大）

四、工作流程

硬件需求

  • 树莓派 4B / 5
    • 推荐至少 4GB RAM，运行 ASR + 说话人识别时比较流畅。
  • 麦克风
    • USB 麦克风（简单易用，例如 TONOR TC-777）
    • 或 麦克风阵列（推荐 ReSpeaker 2/4/6-Mic Array for Raspberry Pi，带有降噪/回声消除）
    • 如果对说话人识别准确率要求高，阵列效果更好。
  • 存储
    • 至少 32GB microSD
    • 或者外接 SSD，ASR 模型会比较大。
  • （可选）扬声器/耳机：做语音交互时需要输出。

软件与模块对应

对应你流程里的各个步骤：

  1. 麦克风输入
    • 用 pyaudio / sounddevice 采集音频流。
    • 设置为 16kHz 单声道 PCM。
  2. Ring Buffer 缓存
    • 可以用 collections.deque 或者 sounddevice.RawInputStream 自带的回调缓存。
    • 缓存 1~3 秒的音频，保证唤醒词检测和语音片段提取。
  3. 唤醒词检测
    • 轻量：Porcupine（低资源占用，树莓派友好）
    • 可训练：Mycroft Precise
    • 自定义词：Vosk + KeywordSpotter
  4. 提取音频片段（1~2秒）

    • 树莓派 4/5 能跑，但建议用轻量模型或外接 USB TPU（Coral USB Accelerator）。
  6. 身份确认
    • 如果 embedding 相似度 > 阈值（如 0.75），则确认说话人。
    • 否则忽略唤醒，避免被陌生人触发。
  7. 完整语音识别（ASR）
    • 用 Vosk（支持离线运行，中文/英文等都有预训练模型）。
    • Vosk-pi 版本可在树莓派直接跑，速度较慢，但可以用小模型（如 vosk-model-small-cn-0.3）。

make和new区别

  • make和new都是golang用来分配内存的內建函数，且在堆上分配内存，make 即分配内存，也初始化内存。new只是将内存清零，并没有初始化内存。
  • make返回的还是引用类型本身；而new返回的是指向类型的指针。
  • make只能用来分配及初始化类型为slice，map，channel的数据；new可以分配任意类型的数据。

数组和切片区别

  • 数组是一个长度固定的数据类型，其长度在定义时就已经确定，内存空间大小是固定；切片是一个长度可变的数据类型，内存空间运行时动态分配，其长度在定义时可以为空，也可以指定一个初始长度 append追加切片
  • 当数组作为函数参数时，函数操作的是数组的一个副本，不会影响原始数组；当切片作为函数参数时，函数操作的是切片的引用，会影响原始切片

GMP协程调度器

内核线程-thread(Machine)，协程(Goroutine)，处理器-G的上下文环境（Processor）

  • G：goroutine协程，基于协程建立的用户态线程
  • M：machine，它直接关联一个os内核线程，用于执行G
  • P：processor处理器，P里面一般会存当前goroutine运行的上下文环境（函数指针，堆栈地址及地址边界），P会对自己管理的goroutine队列做一些调度

调度器策略

  • 全局G队列 - 通过go func()来创建一个goroutine,有两个存储G的队列，一个是局部调度器P的本地队列、一个是全局G队列。新创建的G会先保存在P的本地队列中，如果P的本地队列已经满了就会保存在全局的队列中
  • 复用线程 - work stealing机制和hand off机制（偷别人的去执行，和自己扔掉执行）,G只能运行在M中，一个M必须持有一个P,M与P是1：1的关系,

work stealing机制如果P的本地队列为空，就会想其他的MP组合偷取一个可执行的G来执行
hand off机制当本线程因为G进行系统调用阻塞时，线程释放绑定的P，把P转移给其他空闲的线程执行，此时M1如果长时间阻塞，可能会执行睡眠或销毁

  • 利用并行 - 我们可以使用GOMAXPROCS设置P的数量,这样的话最多有GOMAXPROCS个线程分布在多个CPU上同时运行。GOMAXPROCS也限制了并发的程度，比如GOMAXPROCS = 核数/2，则最多利用了一半的CPU核进行并行
  • 抢占策略 - 1对1模型的调度器，需要等待一个co-routine主动释放后才能轮到下一个进行使用, 如果一个goroutine使用10ms还没执行完，CPU资源就会被其他goroutine所抢占

调度器的生命周期

在了解调度器生命周期之前，我们需要了解两个新的角色M0和G0

M0（跟进程数量绑定，一比一）：

  • 启动程序后编号为0的主线程
  • 在全局变量runtime.m0中，不需要在heap上分配
  • 负责执行初始化操作和启动第一个G
  • 启动第一个G之后，M0就和其他的M一样了

G0（每个M都会有一个G0）：

  • 每次启动一个M，都会第一个创建的gourtine，就是G0
  • G0仅用于负责调度G
  • G0不指向任何可执行的函数
  • 每个M都会有一个自己的G0
  • 在调度或系统调用时会使用M切换到G0，再通过G0进行调度

M0和G0都是放在全局空间的

k8s 主要有 Master 节点和工作节点组成。主节点主要对集群做出全局决策(比如调度)，以及检测和响应集群事件(例如资源不足，自动扩缩容)；从节点负责维护运行的 Pod 并进行通信的网络代理。

  • Mater 节点主要有以下组件：

    • kube-apiserver：负责对外暴露 Kubernetes API。
    • etcd：作为保存 Kubernetes 所有集群数据的后台数据库。
    • kube-scheduler：在适当的时候进行调度决策，让 Pod 在合适的节点上创建运行。
    • kube-controller-manager：负责监控调整调整集群的状态，比如故障检测、自动扩展、滚动更新等

  • Node 节点有以下组件：

    • kubelet：主要负责执行、监控由调度器分配的 Pod，相当于是 Master 在每个 Node 节点上的代理。保证 Pod 的运行状态与目标状态一致。
    • kube-proxy：k8s 在每个节点上的网络代理，负责为 Service 提供集群内部的服务发现和负载均衡。
